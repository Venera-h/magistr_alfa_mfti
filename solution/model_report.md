# Отчет по модели прогнозирования кредитного отклика

## Обзор задачи
Разработка модели для прогнозирования вероятности согласия клиента на кредитное предложение (бизнес-кредит, возобновляемая кредитная линия или овердрафт).

## Данные
- **train_apps.csv**: 145,241 записей, 28 признаков
- **test_apps.csv**: 36,311 записей, 27 признаков
- **Целевая переменная**: `target_value` (0 - отказ, 1 - согласие)

## Анализ данных

### Распределение классов
- Класс 0 (отказ): 136,395 записей (93.9%)
- Класс 1 (согласие): 8,846 записей (6.1%)
- **Соотношение классов**: 1:15.4 (сильный дисбаланс)

### Пропущенные значения
В данных присутствует значительное количество пропущенных значений:
- `loan_rev_max_start_non_fin`: 91.3% пропусков
- `loan_rev_min_start_fin`: 85.9% пропусков
- `overdraft_app_term_max_360`: 96.2% пропусков

### Категориальные признаки
1. `db_group_last` - 9 уникальных значений
2. `fl_adminarea` - 83 уникальных значения (регионы)

## Предобработка данных
1. Удалены идентификаторы: `front_id`, `decision_day`
2. Категориальные признаки закодированы с помощью LabelEncoder
3. Пропущенные значения заполнены медианой
4. Признаки масштабированы с помощью StandardScaler

## Моделирование

### Тестируемые модели
1. **Gradient Boosting (базовый)**
2. **Gradient Boosting (с балансировкой)**
3. **Random Forest (с class_weight='balanced')**

### Результаты на validation set

| Модель | ROC-AUC | Accuracy | Precision | Recall | F1-score |
|--------|---------|----------|-----------|--------|----------|
| Gradient Boosting (базовый) | 0.8163 | 0.9432 | 0.6325 | 0.1605 | 0.2561 |
| **Gradient Boosting (с балансировкой)** | **0.8206** | **0.9435** | **0.6350** | **0.1702** | **0.2684** |
| Random Forest | 0.8064 | 0.8180 | 0.1905 | 0.6122 | 0.2906 |

### Кросс-валидация
- **Средний ROC-AUC**: 0.8258 (+/- 0.0109)
- **Стабильность**: Высокая (низкое стандартное отклонение)

## Рекомендации по использованию модели

### Пороги классификации
1. **Консервативный подход (минимальный риск)**: порог 0.5
   - Только 3.0% клиентов будут классифицированы как "согласие"
   - Высокая precision, низкий recall

2. **Балансированный подход**: порог 0.3
   - 5.9% клиентов будут классифицированы как "согласие"
   - Умеренные precision и recall

3. **Агрессивный подход (максимальный охват)**: порог 0.1
   - 24.0% клиентов будут классифицированы как "согласие"
   - Низкая precision, высокий recall

### Бизнес-интерпретация
Модель показывает, что наиболее важными факторами для согласия на кредит являются:
1. **История кредитных оборотов** (`loan_rev_max_start_non_fin`)
2. **Количество задолженностей за 90 дней** (`cnt_deb_loan_90`)
3. **Последняя сумма кредита** (`loan_amount_last`)
